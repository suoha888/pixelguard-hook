# PixelGuard 中文最终冲刺

这份文档只写最后提交前真正需要照着做的路径。英文完整文档在 `final-30-minute-runbook.md`。

官方截止时间：**2026-05-28 23:59 UTC**，北京时间是 **2026-05-29 07:59**。

## 0. 本地确认

先跑：

```powershell
.\tools\pixelguard-local-audit.ps1
```

应该看到：

```text
PixelGuard local audit passed.
20 tests passed, 0 failed, 0 skipped
```

## 1. 填 `.env`

```powershell
Copy-Item .env.example .env
notepad .env
```

至少填写：

- `PRIVATE_KEY`
- `OKLINK_API_KEY`（可选，由于 OKLink 接口调整，推荐直接手动在 OKX 浏览器网页端验证合约）

主网默认值先不要乱改：

- `XLAYER_MAINNET_RPC=https://rpc.xlayer.tech`
- `V4_POOL_MANAGER=0x360e68faccca8ca495c1b759fd9eee466db9fb32`
- `V4_POSITION_MANAGER=0xcf1eafc6928dc385a342e7c6491d371d2871458b`

然后检查：

```powershell
.\tools\pixelguard-env-check.ps1
.\tools\pixelguard-wallet-check.ps1
```

## 2. 部署顺序

按顺序跑，不要跳：

```powershell
.\tools\pixelguard-deploy.ps1 -Step demoTokens
.\tools\pixelguard-deploy.ps1 -Step router
.\tools\pixelguard-deploy.ps1 -Step hook
.\tools\pixelguard-deploy.ps1 -Step pool
.\tools\pixelguard-deploy.ps1 -Step swap
.\tools\pixelguard-deploy.ps1 -Step largeSwap
.\tools\pixelguard-deploy.ps1 -Step read
.\tools\pixelguard-deploy.ps1 -Step verify
```

*注意：即便没有 API Key，也可以运行最后一步 `-Step verify` 来自动生成并显示手动验证所需的构造函数 Hex 参数。*

每一步都把输出复制到：

```text
docs/submission/deployment-results.md
```

特别注意：

- `demoTokens` 输出 `Recommended TOKEN0` 和 `Recommended TOKEN1`，复制进 `.env` 的 `TOKEN0` / `TOKEN1`。
- `router` 输出 router 地址，复制进 `.env` 的 `V4_SWAP_ROUTER`。
- `hook` 输出 Hook 地址，复制进 `.env` 的 `HOOK_ADDRESS`。
- `read` 输出 latest tokenId、owner、tokenURI，这些是 Demo 视频和表单里的强证据。

## 3. 自动收尾

部署、swap、read、verify 之后跑：

```powershell
.\tools\pixelguard-finalize-submission.ps1
```

它会自动做：

- 汇总 Foundry broadcast
- 尽量回填 `deployment-results.md`
- 生成 OKLink explorer 链接
- 生成 `generated-submit-pack.md`
- 重新生成 release zip
- 检查还有哪些字段没填

如果 X 账号和 X 帖还没做，但你想先检查链上字段：

```powershell
.\tools\pixelguard-finalize-submission.ps1 -AllowSocialPending
```

如果单独重新打包：

```powershell
.\tools\pixelguard-make-release-zip.ps1
.\tools\pixelguard-verify-release-zip.ps1
```

## 4. 最终检查

所有字段填好后：

```powershell
.\tools\pixelguard-readiness.ps1
```

绿色通过之前不要提交。它会检查：

- 地址格式
- 交易哈希格式
- URL 格式
- tick 范围
- tokenURI 是否是 `data:application/json`
- `generated-submit-pack.md` 是否还有 `[FILL]`

## 5. Demo 视频

建议 1-3 分钟，按这个顺序录：

1. 展示本地测试通过。
2. 展示 OKLink 上的 Hook 地址。
3. 展示 PoolId 或 swap tx。
4. 展示 normal swap。
5. 展示 large swap。
6. 展示 `read` 输出里的 receipt tokenId、owner、tokenURI。
7. 打开 `docs/demo/receipt-viewer.html`，粘贴 tokenURI，展示 SVG receipt。

口播稿在：

```text
docs/submission/demo-video-script.md
```

## 8. 最重要的提交字段

Google Form 或 X 帖至少要有：

- Hook 地址
- Hook 验证链接
- V4 PoolId
- normal swap tx
- guarded large swap tx
- OKLink Hook URL
- OKLink swap URL
- GitHub repo URL
- Demo 视频 URL
- X submission post URL

## 9. 应急

如果验证卡住：

先提交 Hook 地址、部署 tx、OKLink 地址页、GitHub repo，并说明 verification pending。能验证最好，但不要为了等验证错过截止时间。

如果 large swap 失败：

优先重新部署 demo tokens 或确认余额，不要临时改合约阈值。

如果主网 gas / 钱包卡住：

只有在你已经确认 testnet v4 `PoolManager`、`PositionManager` 和 router 都可用时才切 testnet。默认建议继续主网。
